工单详情页结构方案

目标状态：已派单。重点比较“查看信息 + 完工录费”的阅读顺序和操作压力。

方案 A
顶部操作带 + 双列详情
已派单
工单编号 / 当前状态 / 操作按钮固定在顶部
客户信息
客户历史订单
施工信息 / 完成工单表单
回访信息

方案 A：双列聚焦

左侧看客户和历史，右侧集中处理施工与完工录费，最像“操作中心”。

方案 B
单列流程式详情
已派单
头部状态条 + 时间线
客户信息
施工信息 / 完成工单表单
客户历史订单
回访信息

方案 B：单列流程

按阅读顺序从上到下展开，最稳，但完工表单会显得更长。

方案 C
摘要先行 + 内嵌编辑区
已派单
顶部摘要条：客户 / 师傅 / 当前动作
工单概览摘要
客户信息
派单信息
下方完整展开：完成工单表单

方案 C：摘要驱动

先让人快速建立全局认知，再进入完工录费，但页面更偏“管理视图”。

我的建议

  • 优先推荐方案 A。
  • 它最符合“详情页是操作中心”的需求定位。
  • 右侧可以把“完成工单”当作当前主任务，减少用户来回扫视。

需要注意

  • 如果右侧表单过长，必须做好分组标题和留白。
  • 左侧信息不能过满，否则会削弱右侧操作焦点。
  • 回访信息在已派单状态下应弱化成占位提示，不应抢主操作。
