### 2. LIKE 查询通配符转义 ✅

**文件**：
- `backend/src/utils/sanitize.js`（新增）
- `backend/src/routes/workOrders.js`

**修复内容**：
- 新增 `escapeLike()` 函数转义 `%` 和 `_`
- 所有 LIKE 查询都使用 `escapeLike()` 处理

```javascript
// backend/src/utils/sanitize.js
function escapeLike(input) {
  if (typeof input !== 'string') return '';
  return input
    .replace(/\\/g, '\\\\')
    .replace(/%/g, '\\%')
    .replace(/_/g, '\\_');
}
```

## 四、剩余问题详细说明

### 问题 1：导出 Excel 全量拉取（未修复）

**文件**：`frontend/src/views/orders/OrderList.vue` 第 422-465 行

**现状**：导出时仍使用 `pageSize: 9999` 一次性拉取所有数据

**影响**：
- 数据量大时请求超时
- 浏览器内存溢出/卡顿
- 后端压力过大

**建议修复**：
- 方案 A（推荐）：后端提供流式导出端点
- 方案 B：前端分批请求，每次 500 条
- 方案 C：限制最大导出数量（如 5000 条），超过时提示用户

---

### 问题 2-3：dashboard 全表加载（未修复）

**文件**：`backend/src/routes/dashboard.js`

**现状**：
- 区域统计：先查出所有 Construction、WorkOrder、CallbackRecord，再在 JS 中聚合
- 师傅排行：查出所有师傅及全部施工记录

**影响**：数据量大时 OOM、响应缓慢

**建议修复**：使用 SQL JOIN + GROUP BY 聚合

---

### 问题 4：工单列表每次查询统计（未修复）

**文件**：`backend/src/routes/workOrders.js` 第 100-120 行

**现状**：每次翻页都重新计算 statusCounts, overdueCount, awaitCallbackCount

**建议修复**：
- 方案 A：统计数据缓存（30秒 TTL）
- 方案 B：通过查询参数 `?includeStats=false` 控制

---

### 问题 5：模板内联函数调用（未修复）
