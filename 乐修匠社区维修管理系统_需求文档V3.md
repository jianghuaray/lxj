## 1. 项目概述

### 1.1 项目背景

乐修匠是一家社区维修服务平台，目前覆盖新城区、未央区、高新区、灞桥区等多个区域，与多个社区有合作关系。接线员每天负责记录客户诉求、对接维修师傅、跟进施工进度并进行回访。

当前所有数据通过Excel表格手工记录（28列大表），存在以下痛点：

- 表格信息密度高，录入和查看效率低，容易出错
- 回访信息大量缺失，流程执行不到位
- 缺乏状态流转机制，全靠人工记忆跟进进度
- 无法快速统计和展示业务数据，领导难以掌握运营状况
- 师傅结算靠手工计算，耗时且容易出错
- 客户投诉缺乏系统化记录和跟踪
- **[新增]** 缺乏独立的客户档案，无法追踪客户历史和偏好

### 1.2 项目目标

构建一套Web端社区维修管理系统，实现工单全流程数字化管理，提升接线员工作效率，为管理层提供数据决策支持。

**核心目标：**

1. **工单全流程线上化**：从客户诉求到回访闭环，每个环节清晰可追踪
2. **数据可视化**：为领导提供丰富的数据看板，掌握运营全貌
3. **师傅结算自动化**：自动计算师傅周期性收款，简化对账流程
4. **投诉管理规范化**：系统化记录和跟踪客户投诉处理
5. **[新增]** **客户档案数字化**：建立独立客户档案，支持客户画像和历史追踪

### 1.3 用户角色

| 角色 | 职责 | 使用场景 |
|------|------|---------|
| **[修改]** 接线员 | 录入客户诉求、派单给师傅、**录入完成信息和费用**、执行回访、处理投诉、**维护客户档案** | 日常办公电脑 |
| 管理员 | 查看数据报表、管理师傅信息、管理服务类型、系统配置、审核结算 | 日常办公电脑 |

## 2. 功能需求

### 2.1 工单管理（核心模块）

#### 2.1.1 工单创建

接线员录入客户维修诉求，字段包括：

- **客户信息**：姓名、联系方式、区域、详细住址
- **诉求信息**：问题分类（下拉选择）、问题描述（文本输入）、来源渠道
- **接线员**：自动记录当前登录用户
- **受理时间**：自动记录当前时间

**问题分类选项**：水电维修、下水疏通、家具门窗、家电维修、家电清洗、测漏防水、开锁换锁、局部翻新

**来源渠道选项**：社区宣传、客户推荐、线上推广、其他

> ### [变更 C1] 2.1.2 工单状态流转

#### 2.1.2 工单状态流转

> **业务背景**：师傅联系客户和上门维修的过程对接线员是黑盒，接线员不需要知道中间进展，只需要在师傅反馈完成后录入结果。**[删除]** 因此去掉了"施工中"状态。

工单状态及流转规则：

- **待派单**：新创建的工单，等待分配师傅
- **[修改]** **已派单**：已指定维修师傅，等待师傅完成维修并反馈（此阶段为黑盒，接线员不操作）
- **[修改]** **已完成**：师傅反馈完成后，接线员录入完成时间和费用信息
- **已回访**：回访完成，工单闭环
- **已取消**：客户取消或无法服务（需填写取消原因）
- **咨询单**：仅咨询，未形成实际服务

**[删除]** ~~施工中：师傅已开始施工~~

**状态流转路径：**

```
[修改] 待派单 → 已派单 → 已完成 → 已回访
         ↓        ↓
        已取消   已取消
       待派单 → 咨询单
```

**[删除]** ~~待派单 → 已派单 → 施工中 → 已完成 → 已回访~~

**关键说明**：
- **[新增]** 已派单 → 已完成：由接线员在工单详情页操作，填写完成时间和费用后确认
- **[新增]** 不需要"施工中"状态，因为师傅的维修过程对接线员不可见
- **[新增]** 不需要已派单超时提醒，等师傅反馈即可

**取消原因选项**：客户主动取消、价格原因取消、师傅无法施工、联系不上客户、其他

> ### [变更 C4] 2.1.4 工单详情（操作中心）

#### 2.1.4 工单详情（操作中心）

> **[新增]** **设计原则**：详情页是所有操作的统一入口，根据工单状态显示不同的操作按钮。

展示工单完整信息，分三个区块卡片式展示：

1. **[修改]** **客户信息区**：基本信息（**[新增]** 可编辑） + 诉求描述 + 接线员备注
2. **施工信息区**：师傅信息、费用明细（总费用、应收服务费、已收服务费、材料成本、抽成比例、楼管激励）、实际维修项目、施工时间
3. **回访信息区**：满意度评分、费用一致性、回访方式、回访人、其他评价、回访时间

**[新增]** 不同状态下的详情页操作：

| 状态 | 可用操作 |
|------|---------|
| 待派单 | [派单] [取消工单] [转为咨询] |
| 已派单 | [完成工单]（展开费用录入表单） [取消工单] |
| 已完成 | [执行回访] |
| 已回访 | 无操作，仅查看 |
| 已取消 | 无操作，仅查看 |
| 咨询单 | 无操作，仅查看 |

**[新增]** 各信息卡片的编辑规则：

> **核心原则**：编辑数据和变更状态是两件事。"操作按钮"触发状态变更+首次录入，"编辑"只修改数据不触发状态变更。

每个信息卡片右上角有"编辑"按钮，点击后该卡片进入编辑模式，出现"保存"/"取消"按钮。

| 区域 | 待派单 | 已派单 | 已完成 | 已回访 | 已取消 | 咨询单 |
|------|--------|--------|--------|--------|--------|--------|
| 客户信息 | ✅ 可编辑 | ✅ 可编辑 | ✅ 可编辑 | ✅ 可编辑 | ✅ 可编辑 | ✅ 可编辑 |
| 施工信息 | 🔒 不可用 | 🔒 不可用（通过"完成工单"首次录入） | ✅ 可编辑 | ✅ 可编辑 | 🔒 不可用 | 🔒 不可用 |
| 回访信息 | 🔒 不可用 | 🔒 不可用 | 🔒 不可用（通过"执行回访"首次录入） | ✅ 可编辑 | 🔒 不可用 | 🔒 不可用 |

- 客户信息：任何状态都能编辑（随时可能需要修正地址、电话等）
- 施工信息：首次通过"完成工单"按钮录入（触发状态变更），之后可随时编辑修改
- 回访信息：首次通过"执行回访"按钮录入（触发状态变更），之后可随时编辑修改
- 未录入数据的区域显示"暂无信息"提示
- 当前系统为单人使用，不需要权限控制

> ### [变更 C5] 2.1.5 客户信息编辑与同步（新增章节）

#### 2.1.5 客户信息编辑与同步

> **[新增]** **数据同步策略**：客户管理是主数据源，工单创建时从客户档案复制信息。工单中修改客户信息时，默认同步更新客户档案。

**[新增]** 工单中编辑客户信息：
- 客户信息区支持随时编辑（姓名、联系方式、区域、地址）
- 保存时弹出同步确认：
  - **同步更新客户档案**（默认勾选）：更新客户管理中该客户的信息，后续新建工单使用新信息
  - **仅更新本工单**：只修改当前工单，不影响客户档案（适用于客户在临时地址维修等场景）

**[新增]** 客户管理中编辑客户信息：
- 只更新客户档案，**不影响已有工单**（历史工单保留当时的真实记录）
- 新建工单时，总是从客户档案读取最新信息

> ### [变更 C6] 2.1.7 完成工单与费用录入（新增章节）

#### 2.1.7 完成工单与费用录入

> **[新增]** **触发场景**：师傅完成维修后反馈给接线员，接线员在工单详情页点击"完成工单"按钮，在详情页内展开费用录入表单。

**[新增]** 操作流程：
1. 接线员进入"已派单"状态的工单详情页
2. 点击"完成工单"按钮
3. 详情页施工信息区变为可编辑状态，显示费用录入表单
4. 填写完成后点击"确认完成"，工单状态变为"已完成"

**[新增]** 费用录入字段：

| 字段 | 说明 | 默认值 |
|------|------|--------|
| 施工完成时间 | 日期时间选择器 | 当前时间，可修改 |
| 维修师傅 | 自动带入（派单时已确定） | 不可修改 |
| 总费用 | 师傅报价，数字输入 | 空 |
| 应收服务费 | 自动计算：总费用 × 师傅抽成比例 | 自动填充，可手动调整 |
| 已收服务费 | 实际收到的金额 | 默认等于应收服务费，可修改 |
| 材料成本 | 数字输入 | 0 |
| 楼管激励 | 数字输入 | 0 |
| 实际维修项目 | 文本框，描述实际做了什么 | 空 |

**收款方式**：师傅代收（客户将费用付给师傅，师傅再交给公司）

**[修改]** 费用逻辑说明：

> - **总费用**：客户付的总金额（师傅代收）
> - **应收服务费** = 总费用 × 抽成比例（公司应得的部分）
> - **实收服务费**：公司实际收到的金额（可能 ≤ 应收服务费）
> - **师傅应得** = 总费用 - 实收服务费
> - **材料成本**：仅记录，由师傅承担，不影响公司利润
> - **楼管激励**：由公司支出
> - **公司利润** = 实收服务费 - 楼管激励

**[新增]** 抽成比例快照锁定机制：

> - 完成工单时，将师傅当前的抽成比例**快照**存入施工记录表（construction.commission_rate）
> - 该工单的服务费按锁定时的比例计算，**后续师傅抽成比例调整不影响历史工单**
> - 师傅管理页面修改抽成比例只影响之后新建的工单
> - 如需调整历史工单的抽成比例，可在工单详情页编辑施工信息

**[新增]** 校验规则：
- 总费用为必填
- 应收服务费不能为负数
- 已收服务费不能大于总费用

### 2.2 回访管理

#### 2.2.1 回访策略

采用选择性回访策略，非每单必回。接线员根据以下情况决定是否回访：

- 费用较高的工单
- 首次服务的客户
- 之前有过投诉的客户
- 管理员指定的重点工单

#### 2.2.2 回访列表

- 显示所有"已完成"但未回访的工单
- 按完成时间排序，优先显示最早的
- 显示等待天数，超过3天标红提醒
- 支持标记"无需回访"并填写原因

#### 2.2.3 执行回访

- 填写回访信息：是否满意、满意度评分（1-5分）、费用是否一致、回访方式（电话/微信/上门）、其他评价
- 回访人自动记录为当前登录用户
- 回访时间自动记录
- 提交后工单状态变为"已回访"
- 若客户不满意，可一键转为投诉工单

### 2.3 师傅管理

#### 2.3.1 师傅信息维护

- 基本信息：姓名、联系方式
- 擅长维修类型（多选：水电维修、下水疏通、家具门窗等）
- **[新增]** 抽成比例（默认30%，可按师傅单独设置，**在师傅卡片上直接展示**）
- 状态（启用/停用）
- 备注

#### 2.3.2 师傅列表

- 查看所有师傅信息
- 支持搜索和筛选（按擅长类型、状态）
- 查看师傅关联的工单历史

#### 2.3.3 师傅对账结算

- 按月自动汇总每位师傅的工单数据
- 结算明细：当月工单列表、每单总费用、应收服务费、已收服务费、材料成本、楼管激励
- 自动计算：当月合计服务费、合计材料成本、应付师傅金额（总费用 - 服务费 - 材料成本）
- 支持导出结算单（Excel格式）
- 管理员可审核确认结算数据

### 2.4 投诉管理

#### 2.4.1 投诉来源

投诉通过两种方式产生：

1. **客户主动投诉**：客户打电话投诉，接线员在系统中记录
2. **回访转投诉**：回访时客户不满意，一键转为投诉

#### 2.4.2 投诉记录

- 关联工单
- 投诉内容描述
- 投诉时间
- 投诉处理人
- 处理状态（待处理/处理中/已解决/已关闭）
- 处理结果描述
- 处理完成时间

#### 2.4.3 投诉列表

- 显示所有投诉记录
- 按状态筛选
- 超过7天未处理的投诉标红

### 2.5 数据看板

#### 2.5.1 概览看板

顶部核心指标卡片（支持切换时间范围：今日/本周/本月/本季度）：

- 工单总数（对比上期增长率）
- 完成率（已完成/总工单）
- 取消率（已取消/总工单）
- 平均满意度评分
- 总营收（应收服务费合计）
- 待回访工单数
- 待处理投诉数

#### 2.5.2 趋势图表

- **工单量趋势**：按日/周/月展示工单数量变化（折线图）
- **维修类型分布**：各类型维修工单占比（饼图/环形图）
- **区域分布**：各区域工单数量对比（柱状图）
- **师傅绩效排行**：接单量、完成率、满意度综合排行（柱状图）
- **满意度趋势**：客户满意度评分变化趋势（折线图）
- **营收趋势**：服务费收入变化趋势（折线图+柱状图）

#### 2.5.3 社区维度分析

- 支持自定义导出字段和时间范围

### 2.6 自动提醒

#### 2.6.1 提醒规则

- 工单完成后3天未回访 → 提醒接线员（标红）

| 材料成本（记录） | 仅记录，师傅承担 | Σ 材料成本 |
| 楼管激励 | 公司支出 | Σ 楼管激励 |
| **公司利润** | **公司真正赚的** | **Σ(实收服务费 - 楼管激励)** |
