## 师傅删除逻辑修复

### 问题
- 后端删除接口（technicians.js 第388-402行）是**软删除**：只设 `status: 0`，记录仍在数据库
- 前端显示"删除成功"，但列表刷新后记录还在（只是状态变"停用"）

### 修复（backend/src/routes/technicians.js）
- `router.delete('/:id')` 中的 `await technician.update({ status: 0 })` 改为 `await technician.destroy()`
- 彻底删除记录，而非软删除

### 注意
- 如果师傅有关联的 Construction 记录，可能因外键约束删除失败
- 后端入口文件是 `src/app.js`（非 `src/index.js`）
