# 2026-05-04 工作记录

## 代码审查报告分析 + 批量修复

基于两份代码审查报告（代码审查报告.md 和 code-review-report.md），完成以下修复：

### 后端修复
1. **app.js** - CORS 限制为前端域名（process.env.FRONTEND_URL || localhost:5173）；请求体限制 1mb；启动时校验 JWT_SECRET（缺失则 process.exit(1)）
2. **routes/auth.js** - 登录接口加 express-rate-limit（15分钟内最多10次）；修改密码接口增加密码强度校验（>=8位含字母数字）
3. **middleware/auth.js** - authAdmin 加防御性 req.user 检查
4. **routes/workOrders.js** - 引入 sequelize 实例；创建工单、assign派单、dispatch派单、callback回访 四个操作全部加事务；工单详情 include CallbackRecord 时加 include User（修复 callbackByName 始终为 null 的 bug）
5. **routes/callbacks.js** - 引入 sequelize 实例；/:id/complete 和 /:orderId 两条回访路由加事务；GET /pending 和 status=pending 分支修复为排除已有回访记录的工单（用 '$callback.id$': null + subQuery: false）
6. **routes/customers.js** - DELETE /:id 加 authAdmin 中间件，仅管理员可删除客户
7. **routes/users.js** - 重置密码不再默认 123456，必须传 newPassword 且满足强度要求
8. **models/** - WorkOrder、Construction、CallbackRecord、OperationLog、Settings 五个模型添加 indexes 配置

### 前端修复
9. **views/Login.vue** - 删除登录成功时泄露 token 的 console.log
10. **views/TechnicianAdd.vue** - 删除保存时泄露表单数据的 console.log
11. **utils/api.js** - 401 处理改为调用 authStore.logout() 而非直接操作 localStorage（同步 Pinia Store 状态）
12. **views/Dashboard.vue** - handleResize 加 debounce(200ms) 防抖，onUnmounted 时清理 debounce timer

### 注意事项
- express-rate-limit 已通过 npm install 安装到 backend/
